## 2025-02-23 - Fast string literal matching for binary data over Regex
**Learning:** Using `re.search(rb"pattern", raw, re.I)` on large chunks of raw binary data is notably slower than using Python's highly optimized substring matching `b"pattern" in raw.lower()`. Benchmarks showed a 4.6x - 7.5x speedup by avoiding the regex engine entirely for exact string matching in hot paths like text extraction routines.
**Action:** When scanning large raw binary strings (like `raw` PDF bytes) for exact string matches without needing complex pattern rules, always prefer `.lower()` combined with the `in` operator over `re.search` with `re.I`. Additionally, remove redundant regex searches if the substring is already proven to exist via a pre-check cache.
