## 2024-07-12 - Tkinter Bridge Optimization in Report Generators
**Learning:** In `src/exporter.py` and `src/export_logic.py`, interacting with the Tkinter UI tree (via `tree_get_children` and `tree_item`) inside a loop over dataset rows creates severe O(N^2) lag due to IPC overhead. A single tag lookup iteration was taking 0.2290s vs 0.0017s with a pre-computed map.
**Action:** When correlating data exports with UI elements in Python, always pre-map the required UI states (like tags or classes) into a standard Python dictionary before entering the main loop to ensure O(1) lookups and bypass the C-level bridge bottleneck entirely.
