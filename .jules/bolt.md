## 2024-XX-XX - Precompiling regular expressions in advanced forensics
**Learning:** Found multiple usages of `re.findall(r'...', txt)` and `re.search(r'...', txt)` inside functions like `detect_emails_and_urls`, `detect_unc_paths`, `detect_encryption_status`, `detect_hidden_text_patterns`, `detect_attachments`, `detect_3d_and_multimedia`, `detect_temporal_anomalies`, `detect_pdf_a_compliance`, `detect_text_operator_anomalies`, and `detect_timestamp_mismatches` in `src/advanced_forensics.py`. While Python caches recently used regexes, explicitly pre-compiling 22+ regex patterns at the module level completely eliminates the cache lookup overhead, which is significant when running forensics over thousands of PDF pages and text chunks.
**Action:** Extract all inline regex strings into module-level `re.compile()` constants.
