
## 2024-05-30 - Using .lower() string prechecks for large regex evaluations
**Learning:** `re.I` with regex functions `re.search` and `re.findall` scale linearly with text size, taking heavy performance penalties on missing values against very large strings (e.g., extracted PDF contents). Assigning a cached `.lower()` string and running `in` on that first can bypass expensive regex computations when strings aren't matched. A `1.6x` overall improvement was found within `detect_indicators` because multiple text-heavy regex matchings were bypassed on average. Note that this caching does not replace regex matching as it will match simple sequences, but acts to fail fast when `target_string` is definitely not in the string being searched.
**Action:** When running multiple consecutive `re.search` or `re.findall` over large contents using case-insensitivity (`re.I`), construct a lowercased cache (e.g. `txt_lower = txt.lower()`) and precede regex logic with an `if "pattern" in txt_lower:` check where appropriate. Use this same object to substitute logic using `.lower().count()`.
